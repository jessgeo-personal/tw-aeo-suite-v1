1. Understanding AI Crawler Blocking

Why Sites Block Bots

Websites typically block automated bots to prevent:

  • Malicious scraping and content theft
  • DDoS attacks and resource overload
  • Spam and fraudulent activity
  • Competitive intelligence gathering

Why AI Crawlers Are Different

AI search engines like ChatGPT, Claude, Perplexity, and Google AI Overviews use specialized crawlers to:

  • Index content for AI-powered search responses[1]
  • Generate accurate, cited answers to user queries[2]
  • Surface your content in AI-generated summaries[3]

⚠️ The Cost of Blocking AI Crawlers

According to recent studies, 60% of all searches now use AI-powered tools[4]. If AI crawlers can't access your site, you're invisible to this majority of searchers. Citation rates for accessible sites are 3.2x higher than those blocking AI bots[5].

2. Identifying Block Types

Cloudflare Challenge Pages

Symptoms: Page displays "Checking your browser" or "Please wait while we verify..."

Impact: CRITICAL - Blocks all AI crawlers by default

HTTP 403 Forbidden

Symptoms: Server returns 403 status code

Impact: CRITICAL - Complete access denial

HTTP 429 Rate Limiting

Symptoms: "Too Many Requests" error

Impact: HIGH - May prevent full indexing

WAF Blocking (Imperva, Incapsula)

Symptoms: Page shows WAF error message

Impact: CRITICAL - Enterprise-level blocking

reCAPTCHA Challenges

Symptoms: "I'm not a robot" checkbox appears

Impact: CRITICAL - Bots cannot solve CAPTCHAs

3. Cloudflare Solutions

Cloudflare powers over 20% of the web[6] and is the most common cause of AI crawler blocking.

RECOMMENDED

Method 1: Enable Verified Bots

This method automatically allows all verified AI crawlers recognized by Cloudflare.

Step-by-Step Instructions:

  1. 1 Log into Cloudflare Dashboard → Select your domain
  2. 2 Navigate to Security → Bots
  3. 3 Under "Bot Fight Mode" section, enable "Allow verified bots" toggle
  4. 4 This automatically allows: GPTBot, ClaudeBot, Google-Extended, PerplexityBot, Bingbot, and other verified crawlers[7]

⏱️ Takes effect: 2-3 minutes

✅ Allows: All verified AI crawlers + legitimate bots

Method 2: Custom WAF Rule

For more control, create a custom rule to allow specific user agents.

Step-by-Step Instructions:

  1. 1 Go to Security → WAF → Custom rules
  2. 2 Click "Create rule"
  3. 3 Rule name: "Allow AI Crawlers and AEO Bot"
  4. 4 Expression (copy/paste this): 
(http.user_agent contains "GPTBot") or
(http.user_agent contains "ClaudeBot") or
(http.user_agent contains "Google-Extended") or
(http.user_agent contains "PerplexityBot") or
(http.user_agent contains "Bingbot") or
(http.user_agent contains "AIOptimizeBot")
  1. 5 Action: Skip → Select "All remaining custom rules"
  2. 6 Click "Deploy"

Method 3: IP Allowlist (AEO Analyzer Only)

For allowing only the AEO analyzer bot without changing broader security settings.

Step-by-Step Instructions:

  1. 1 Go to Security → WAF → Tools
  2. 2 Under "IP Access Rules", click "Add"
  3. 3 Contact support@thatworkx.com for our bot's IP address
  4. 4 Action: Allow
  5. 5 Zone: This website

4. Other Platform Solutions

Imperva/Incapsula WAF

  1. Log into Imperva console
  2. Navigate to Security → Whitelist
  3. Add user agents: GPTBot, ClaudeBot, Google-Extended, PerplexityBot, Bingbot
  4. Apply changes and wait 5-10 minutes for propagation

WordPress Security Plugins

Common plugins like Wordfence, iThemes Security:

  1. Go to plugin settings → Firewall or Blocking
  2. Find Whitelist or Allowlist section
  3. Add AI crawler user agents to allowlist
  4. Save changes

7. Common Mistakes

❌ Blocking All Bots

Don't use blanket "block all automated traffic" rules. This blocks legitimate AI crawlers along with malicious bots.

❌ Multiple Blocking Layers

Check all blocking points: WAF, robots.txt, .htaccess, WordPress plugins. One allowlist doesn't help if another layer blocks.

❌ Not Testing Changes

Always verify your changes worked by running a new AEO analysis or checking server logs.

❌ Forgetting Subdomains

Apply AI crawler allowlisting to all relevant subdomains (www, blog, docs, etc.).
